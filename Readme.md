# Manual Testing Portfolio – Trainline Web Application

This repository contains manual test cases created as part of my transition into a Junior QA / Test Engineer role. The test cases focus on validating core user authentication functionality on the Trainline web application.

## Application Under Test
Trainline – Web Application

## Scope of Testing
The following areas were tested:
- User login functionality
- Input validation and error handling
- Authentication failure scenarios
- Password recovery (Forgot Password)

## Testing Type
- Manual functional testing
- Exploratory testing
- Negative testing

## Test Artifacts
- Login and signup test cases documented in Markdown
- Positive and negative test scenarios
- Validation and error-handling checks

## Assumptions & Notes
- A test (dummy) user account was created for testing purposes
- Real credentials are not stored in this repository
- Placeholders are used for sensitive information such as emails and passwords

## Tools Used
- Manual testing techniques
- GitHub for version control and documentation

## Status
In Progress – additional scenarios and features will be added
